Lua内建的IO库提供了两种不同的处理文件的方式，它们分别是隐式文件描述和显示文件描述。其中：

- 隐式文件描述，是指首先设置一个默认的输入或输出文件，然后通过`io.xxx()`的方式读写这个文件。其中，io就是io库
- 显示文件描述，是指首先获取一个文件的句柄（即打开文件），然后通过`filename:xxx()`的方式读写这个文件。其中，filename是一个文件被打开后的句柄。

需要注意的是，在 OpenResty 的上下文中，文件操作对事件循环会产生阻塞效应。 OpenResty 比较擅长的是高并发网络处理，在这个环境中，任何文件的操作都会阻塞其他并行执行的请求。因此，在OpenResty的实际应用中，项目应尽可能让网络处理部分、文件 I/0 操作部分相互独立，不要揉和在一起。 



## 打开文件

### io.open

```lua
io.oopen(filename [,mode])
```

以指定的模式（mode），打开一个名为filename的文件。打开成功则返回文件句柄，打开失败则返回nil和错误信息。open函数支持的模式如下：

| 模式 | 含义（注：连续是指在现有数据的尾部追加） | 当文件不存在时    |
| ---- | ---------------------------------------- | ----------------- |
| 'r'  | 只读模式（默认）                         | 返回nil和错误信息 |
| 'w'  | 只写模式。以覆盖的方式，连续写           | 创建文件          |
| 'a'  | 只写模式。以追加的方式，不换行并连续写   | 创建文件          |
| 'r+' | 读写模式。以追加的方式，不换行并连续写   | 返回nil和错误信息 |
| 'w+' | 读写模式。以覆盖的方式，连续写           | 创建文件          |
| 'a+' | 读写模式。以追加的方式，不换行并连续写   | 创建文件          |

以上这些模式用于对文本文件的读写。我们还可以在这些模式字符串的后面加一个b来读写二进制文件。

> "w"和“wb”的区别：
>
> - "w"表示文本文件。 Linux 的文件系统认为 0x0A 为文本文件的换行符，Windows 的文件系统认为 0x0D0A 为文本文件的换行符。Windows 为了兼容其他文件系统（如从 Linux 拷贝来的文件）， 在写文件时会在文件中 0x0A 的前面加上 0x0D。 因此， 使用 "w"，其属性要看所在的平台。
>
> - "wb" 表示二进制文件。文件系统会按纯粹的二进制格式进行写操作，因此也就不存在格式转换的问题。
>
> 值得一提的是，Linux 下 "w" 和 "wb" 没有区别



### io.input

```lua
io.input([filename])
```

该函数的调用场景如下：

- 当使用一个文件名调用时，以只读模式打开这个文件，并将该文件的句柄设置为默认的输入文件；
- 当使用一个文件句柄调用时，将这个文件句柄设置为默认的输入文件。
- 当不使用参数调用时，返回默认的输入文件的句柄。

需要注意的是：

- 如果filename不存在，则抛异常。
- 在默认情况下，默认的输入文件为标准输入。即io.input()返回的是标准输入。



### io.output

```lua
io.output([filename])
```

该函数类似于io.input， 但该函数操作的是默认输出文件。其调用场景如下：

- 当使用一个文件名调用时，以覆盖只写模式打开这个文件，并将该文件的句柄设置为默认输出文件；
- 当使用一个文件句柄调用时，将这个文件句柄设置为默认输出文件。
- 当不使用参数调用时，返回默认输出文件的句柄。

需要注意的是：

- 如果filename不存在，则创建文件。
- 在默认情况下，默认输出文件为标准输出。即io.output()返回的是标准输出。



### io.type

```lua
io.type(obj)
```

 检测 obj 是否为一个可用的文件句柄。如果 obj 是一个打开的文件句柄，则返回字符串"file"。 如果 obj 是一个已关闭的文件句柄，则返回字符串"closed file" 。如果 obj 不是一个文件句柄，则返回 nil。 



## 读文件

### filename:lines

```lua
filename:lines()
```

返回一个可迭代函数。每次迭代这个可迭代函数将获得filename中的一行内容，当迭代到文件末尾时，将返回 nil，但不关闭文件。 



### io.lines

```lua
io.lines([filename])
```

以只读模式打开文件并返回一个可迭代函数。每次迭代这个可迭代函数将获得文件中的一行内容， 当迭代到文件末尾时，将返回 nil，并自动关闭文件。 

当不指定filename时，io.lines()等价于io.input():lines()即读取默认输入文件， 结束时不关闭该文件。

注：如果filename不存在，则抛异常。



### filename:read

```lua
filename:read(...)
```

按指定的格式读取一个文件。每个格式将返回一个字符串或数字, 如果不能正确读取将返回 nil，若没有指定格式将按行进行读取。 支持的格式如下：

| 格式   | 含义                                                         |
| ------ | ------------------------------------------------------------ |
| "*n"   | 读取一个数字                                                 |
| "*a"   | 从当前位置读取整个文件。若当前位置为文件末尾，则返回空字符串 |
| "*l"   | 读取下一行的内容。若为文件末尾，则返回nil。(默认格式)        |
| number | 读取指定字节数的字符。如果number为0,则返回空字符串。若为文件末尾，,则返回nil。 |



### io.read

```lua
io.read(...)
```

 相当于 io.input():read(...)，即读取默认输入文件。



## 写文件

### filename:write

```lua
file:write(...)
```

将每个参数的值写入到filename。参数必须为字符串或数字，若要输出其它值，则需通过 tostring或string.format 进行转换。 



### io.write

```lua
io.write(...)
```



相当于 io.output():write(...)，即写入到默认的输出文件。



### filename:flush

```lua
filename:flush()
```

 把缓冲区中的所有数据写入到 filename 。 



### io.flush

```lua
io.flush()
```

 把缓冲区中的所有数据写入到默认输出文件。 



## 关闭文件

### filename:close

```lua
filename:close()
```

关闭名为filename的文件。 

需要注意的是，当文件句柄被垃圾回收后，文件将自动关闭。句柄将变为一个不可预知的值。 



### io.close

```lua
io.close([filename])
```

关闭文件。没有指定filename时，关闭默认输出文件。 



## seek & setvbuf

### filename:seek

```lua
filename:seek([whence][, offset])
```

设置和获取当前文件指针的位置，成功则返回最终的文件指针位置(按字节，相对于文件开头)，失败则返回nil和错误信息。其中，whence 默认为 "cur"，offset 默认为 0 。 参数 whence可以为： 

| whence | 含义                     |
| ------ | ------------------------ |
| “set”  | 文件开头                 |
| "cur"  | 当前文件指针的位置(默认) |
| "end"  | 文件结束                 |



### filename:setvbuf

 ```lua
filename:setvbuf(mode[, size])
 ```

设置输出文件的缓冲模式。可用模式如下： 

| 模式   | 含义                                                      |
| ------ | --------------------------------------------------------- |
| "no"   | 没有缓冲，即直接输出                                      |
| "full" | 全缓冲，即当缓冲满后才进行输出操作(也可调用flush马上输出) |
| "line" | 以行为单位，进行输出                                      |

使用“full”或“line”模式时，可以通过size指定缓冲的大小（按字节），忽略 size 将自动调整为最佳的大小。



## 小结

在Lua中操作一个文件时，一般会用到如下几个函数：

- io.open、io.type
- filename:lines
- filename:write、filename:flush
- filename:close
- filename:servbuf、filename:seek
