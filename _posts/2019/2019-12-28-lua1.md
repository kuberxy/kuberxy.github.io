Lua，在葡萄牙语里代表美丽的月亮。事实证明它没有糟蹋这个优美的单词，Lua 语言正如它的名字所预示的那样成为了一门简洁、优雅且富有乐趣的语言。

## 简介

### Lua

Lua 从诞生之初就是作为一门可嵌入、可扩展的轻量级脚本语言来设计的，因此它一直遵从着“简单、小巧、可移植、快速”的原则，官方实现完全采用ANSI C 编写，它能以 C 程序库的形式嵌入到宿主程序中。LuaJIT 2 和标准 Lua 5.1解释器采用的是著名的 MIT 许可协议。

正是由于上述特点， Lua 在游戏开发、机器人控制、分布式应用、图像处理、生物信息学等各种各样的领域中得到了越来越广泛的应用。其中尤以游戏开发为最，许多著名的游戏，比如 Escape from Monkey Island、World of Warcraft、大话西游，都采用了 Lua 来配合引擎完成数据描述、配置管理和逻辑控制等任务。即使像 Redis 这样高性能的内存键值数据库也提供了内嵌用户 Lua 脚本的官方支持。



### Lua vs LuaJIT

Lua 非常高效，它运行得比许多其它脚本(如Perl、Python、Ruby)都快，这点在第三方的独立测评中得到了证实。尽管如此，仍然会有人不满足，他们总觉得“嗯，还不够快!”。LuaJIT 就是一个为了进一步榨出一些速度的尝试，它利用即时编译（Just-inTime）技术把 Lua 代码编译成本地机器码后交由 CPU 直接执行。

LuaJIT 2 的测评报告表明，在数值运算、循环与函数调用、协程切换、字符串操作等许多方面它的加速效果都很显著。凭借着 FFI 特性，LuaJIT 2 在那些需要频繁地调用外部 C/C++代码的场景，也要比标准 Lua 解释器快很多。目前 LuaJIT 2 已经支持包括 i386、x86_64、ARM、PowerPC 以及 MIPS 等多种不同的体系结构。

> LuaJIT = Lua解释器 + 即时编译器

LuaJIT 是采用 C 和汇编语言编写的 Lua 解释器与即时编译器。LuaJIT 被设计成全兼容标准的 Lua 5.1 语言，同时可选地支持 Lua 5.2 和 Lua 5.3 中的一些不破坏向后兼容性的有用特性。因此，标准 Lua 语言的代码可以不加修改地运行在 LuaJIT之上。

LuaJIT 和标准 Lua 解释器的一大区别是，LuaJIT 的执行速度，即使是其汇编编写的 Lua 解释器，也要比标准 Lua 5.1 解释器快很多，可以说是一个高效的Lua 实现。另一个区别是，LuaJIT 支持比标准 Lua 5.1 语言更多的基本原语和特性，因此功能上也要更加强大。

### 定义

在Lua 中，变量默认就是全局变量，只有使用local关键字显示声明的变量才是局部变量。

```lua
g_var = 1         -- global var
local l_var = 2   -- local var
```



### 作用域

局部变量的生命周期是有限的，它的作用范围仅限于定义它的block。

> block即块，一个块可以是一个控制结构的执行体（如if、for、while等），或者是一个函数的函数体，还可以是一个程序块（也称为chunk）。

我们先来看一个简单示例：

```lua
a = 1
local b = 2

if a then
    c = 3
    local d = 4
    print(a,b,c,d)
end

function func()
    print(a,b,c,d)
end


func()

print(a,b,c,d)
```

运行结果：

```lua
1	2	3	4
1	2	3	nil
1	2	3	nil
```

因为，变量d是if结构中的局部变量，所以它的作用范围只在if结构内。而变量b是全局作用域中的局部变量，因此它的作用范围整个程序。



下面我们再来看一个示例：

```lua
x = 10
local i = 1            -- 程序块中的局部变量i

print("while: ")
while i < x do
    local x = i * 2    -- while循环体中的局部变量x
    print(i,x)         -- output： 2, 4, 6, 8, ...
    i = i + 1
end

print("\nif: ")
if i > 10 then
    local x            -- if块中的局部变量 x
    x = 20
    print(x + 2)       -- 如果i > 10 将会打印 22。此处的 x 是局部变量
else
    print(i,x)         -- 打印10 10，这里的 x 是全局变量
end

print("\nglobal: ")
print(i,x)             -- 打印10 10
```

运行结果：

```lua

1	2
2	4
3	6
4	8
5	10
6	12
7	14
8	16
9	18

if: 
10	10

global: 
10	10
```



### 使用局部变量的好处

局部变量有如下好处：

- 局部变量可以避免因为命名问题污染了全局环境。

- local 变量的访问比全局变量更快

- 由于局部变量出了作用域之后生命周期结束，这样可以被垃圾回收器及时释放

因此，我们在一些地方会看到这样的使用方式：

```lua
local print = print
```

在Lua中，应该尽量让“定义变量的语句靠近使用变量的语句”，这也是一种良好的编程风格。

> 在 C 这样的语言中，会强制程序员在一个块（或一个过程）的起始处声明所有的局部变量，所以有些程序员认为在一个块的中间使用声明语句是一种不良好地习惯。
>
> 实际上，在需要时才声明变量并且赋予有意义的初值，这样可以提高代码的可读性。

### 关系运算符 - <、>、<=、>=、==、~=

```lua

```

需要注意的是，在使用“==”做等于判断时，对于table、userdate和函数，Lua是作引用比较的。也就是说，只有当两个变量引用同一个对象时，才认为它们相等。我们看下面的例子：

```lua
local a = {x=1,y=0}
local b = {x=1,y=0}

    print("a==b")
else
    print("a~=b")
end

print(a)
print(b)


local c = a
if c == a then
    print("c==a")
else
    print("c~=a")
end

print(c)
print(a)


print({x=1,y=0})


local x,y= 'test','test'
if x == y then
    print("x==y")
else
    print("x~=y")
end
```

在Lua中，字符串总是会被“内化”，即对于内容相同的字符串只会保存一份。因此，对于值为字符串的变量来说，它们之间的相等性比较可以简化为其内部存储地址的比较。这也意味着，在Lua中字符串的相等性比较总是为O(1)，而在其他编程语言中，字符串的相等性比较通常为O(n)，即需要逐个字节（或按若干个连续字节）进行比较。



### 逻辑运算符 - and、or、not

在Lua中，and和or是不同于其他语言的。在许多编程语言中，and和or运算的返回值总是1或0，即真或假。而Lua中and和or的执行过程是这样的：

- a and b，当a为假，则返回a；当a为真，则返回b。
- a or b，当a为假，则返回b；当a为真，则返回a。

注：在Lua中，所有的逻辑操作将 false 和 nil 视作假，其他任何值视作真。对于 and 和 or，“短路求值”；对于 not，永远只返回 true 或者 false。

### 字符串连接 - ..

在Lua中，连接两个字符串，可以使用操作符“..”（两个点）。如果操作数是数字类型，则Lua会先将其转换成字符串类型。需要注意的是，连接操作会创建一个新字符串，而不会改变原操作数。

此外，连接两个字符串，还可以使用string库函数string.format。

```lua
print("Hello " .. "World")
print(0 .. 1)
print(string.format("%s %s", "hello","world"))
print(string.format("%d %s %.2f", "123","word",1.21))
print(string.format("%s-%s-%s",2008,08,08))
print(string.format("%d:%d:%d",10,30,20))
```

由于 Lua 字符串本质上是只读的，因此字符串连接运算符几乎总会创建一个新的（更大的）字符串。这意味着如果有很多这样的连接操作（比如，在循环中使用 .. 来拼接字符串），则性能损耗会非常大。在这种情况下，推荐使用 table 和 table.concat() 来进行很多字符串的拼接，例如：

```lua
local pieces = {}
for i, elem in ipairs(my_list) do
    pieces[i] = my_process(elem)
end

local res = table.concat(pieces)
```

当然，在这个例子中还可以使用LuaJIT独有的table.new来恰当地初始化pieces表的空间，以避免该表的动态生长。

### for

在Lua中，for 语句有两种形式：数字 for（numeric for）和泛型 for（generic for）。

#### 数字for

##### 语法

```lua
for var = begin, finish, step do
    -- body

```

关于数字 for，需要关注以下几点： 

- var 从 begin 变化到 finish。每次变化时，var都以 step 作为步长递增。其中，step 是可选的，默认为 1。
- begin、finish、step 三个表达式只会在循环开始时执行一次，即begin、finish、step在循环开始时就确定了。
- var 的作用域仅在 for 循环内，如果需要在外面控制，则需将值赋给一个新的变量。
- 循环过程中不要改变控制变量的值，这会带来不可预知的影响。
- 如果不想给循环设置上限的话，可以使用常量 math.huge，即使用for实现无限循环。

#### 泛型for

泛型 for，是指通过一个迭代器（iterator）函数来遍历一个对象。



##### 语法

```lua
for i[,j,k] in iterator do
    -- body
end
```

泛型 for 循环与数字型 for 循环有两个相同点： 

（1）循环变量是循环体中的局部变量； 

（2）决不应该对循环变量作任何赋值操作。



##### 示例

在Lua 的标准库中提供了一个 ipairs函数，它就是一个迭代器函数可以用来遍历数组。在每次循环时，它会返回两个值，第一个是索引值，第二个是该索引在数组中对应的元素值。

```lua
local a = {"a", "b", "c", "d"}

for k, v in ipairs(a) do
    print("index:", k, " value:", v)
end
```

运行结果：

```lua
index:  1  value: a
index:  2  value: b
index:  3  value: c
index:  4  value: d
```

从外观上看泛型 for 比较简单，但其实它是非常强大的。通过使用不同的迭代器函数，几乎可以遍历所有的东西， 而且写出的代码极具可读性。标准库中提供了几种迭代器函数，包括用于按行迭代文件的（io.lines）、 迭代 table 元素的（pairs）、迭代数组元素的（ipairs）、迭代字符串中单词的（string.gmatch）等。



对于泛型 for 的使用，再来看一个更具体的示例。假设有这样一个 table，它的内容是一周中每天的名称：

```lua
local days = {"Sunday", "Monday", "Tuesday", "Wednesday","Thursday", "Friday", "Saturday"}
```

现在要将名称转换成它在一周中的位置。比如，它在Sunday一周中的位置为1。为此，需要根据给定的名称来搜索这个 table。然而在 Lua 中，通常更有效的方法是创建一个“逆向 table”。例如这个逆向 table 叫 revDays，它以一周中每天的名称作为索引，位置数字作为值：

```lua
local revDays = {
    ["Sunday"] = 1,
    ["Monday"] = 2,
    ["Tuesday"] = 3,
    ["Wednesday"] = 4,
    ["Thursday"] = 5,
    ["Friday"] = 6,
    ["Saturday"] = 7
}
```

接下来，要找出一个名称所对应的需要，只需用名字来索引这个 reverse table 即可：

```lua
local x = "Tuesday"
print(revDays[x]) -- 3
```

当然，这个逆向 table不必手动声明，而是通过原来的 table 自动地构造出这个逆向table：

```lua
local days = {"Sunday", "Monday", "Tuesday", "Wednesday","Thursday", "Friday", "Saturday"}
  
local revDays = {}
for k,v in pairs(days) do
    revDays[v] = k
end

for k,v in pairs(revDays) do
    print("k:", k, " v:", v)
end
```

这个循环会为每个元素进行赋值，其中变量 k 为 key(1、2、...)，变量 v 为value("Sunday"、"Monday"、...)。

> 值得一提的是，在 LuaJIT 2.1 中， 内建函数ipairs() 是可以被 JIT 编译的，而pairs()  则只能被解释执行。因此在性能敏感的场景，应当合理安排数据结构，避免对哈希表进行遍历。
>
> 事实上，即使未来  pairs  可以被 JIT 编译，哈希表的遍历本身也不会有数组遍历那么高效，毕竟哈希表就不是为遍历而设计的数据结构。



### while

while语句，在表达式值为假（即 false 或 nil）时结束循环。此外，可以使用 [break](https://moonbingbing.gitbooks.io/openresty-best-practices/lua/break.html) 语言提前跳出循环。

#### 语法

```lua
while 表达式 do
    -- body
end
```



#### 示例

求 1 + 2 + 3 + 4 + 5 的结果：

```lua
local i = 1
local s = 0

while true do
    s = s + i 
    i = i + 1
    if i > 5 then
	print(s)
        break
    end
end
```

值得一提的是，Lua 并没有像许多其他语言那样提供类似 continue 这样的控制语句，来实现立即进入下一个循环迭代（如果有的话）。因此，我们需要仔细地安排循环体里的分支，以避免这样的需求。

### nil（空）

nil是一种类型，Lua 将 nil 用于表示“无效值”。一个变量在第一次赋值前，它的默认值是 nil，将一个全局变量赋值为nil就等同于删除这个变量。

```lua
local a
print(a)
print(b)

b = 100
print(b)
```

运行结果：

```lua
nil
nil
100
```

在OpenResty 提供的Lua 接口中有一种特殊的空值，即 ngx.null，它用来表示不同于 nil 的“空值”。



### boolean（布尔）

布尔类型，即 true/false；在Lua 中 nil 和 false 为假，其它所有值均为真。比如， 0 和空字符串也为真。

```lua
local a = nil
local b = false
local c = true
local d = 0
local e = ''
local f = 1

print(a,b,c,d,e,f)
print(not a,not b,not c,not d,not e,not f)
```

运行结果：

```lua
nil	    false	true	0		        1
true	true	false	false	false	false
```



### number（数字）

Number 类型用于表示实数，和 C/C++ 里面的 double 类型类似。Lua中的Number类型也支持使用数学函数 math.ceil（向上取整）和math.floor（向下取整）进行取整操作。

```lua
local a = 3.99
local b = 98.01

print(math.ceil(a))
print(math.floor(a))

print(math.ceil(b))
print(math.floor(b))
```

运行结果：

```
4
3
99
98
```

Lua 的 number 类型就是用双精度浮点数来实现的。

值得一提的是，LuaJIT 支持所谓的“dual-number”（双数）模式，即 LuaJIT 会根据上下文用整型来存储整数，而用双精度浮点数来存放浮点数。另外，LuaJIT 还支持“长整型”的大整数（在 x86_64 体系结构上则是 64 位整数）。例如：

```lua
> print(9223372036854775807LL - 1)    
9223372036854775806LL
```



### string（字符串）

在Lua 中有三种方式表示字符串:

- 使用一对匹配的单引号。如，'hello'

- 使用一对匹配的双引号。如，"hello"

- 使用一对匹配的方括号。如，[[hello]]



我们看如下示例：

```lua
local str1 = 'hello world'
local str2 = "hello lua"
local str3 = [["add\name", 'hello']]
local str4 = [=[string have a [[]].]=]

print(str1)
print(str2)
print(str3)
print(str4)
```

运行结果：

```lua
hello world
hello lua
"add\name", 'hello'
string have a [[]].
```

我们可以在两个正方括号（即`[[`）和两个反括号（即`]]`）的中间插入n个等号，来定义一个n级的长括号。比如，0级的长括号写作“[[string]]”，一级的长括号写作“[=[ sting]=]”，二级的长括号写作“[==[string]==]”，以此类推，就能写出一个n级的长括号。

```lua
local s1 = [["add\tname\nadd\tname"]]
local s2 = [[a
b]]
local s3 = [[ [] ]]
-- local s4 = [[ [[]] ]]
local s5 = [[ [=[]=] ]]
local s6 = [[ [==[]==] ]]
local s7 = [=[ [[]] ]=]

print(s1)
print(s2)
print(s3)
print(s4)
print(s5)
print(s6)
print(s7)
```

运行结果：

```lua
"add\tname\nadd\tname"
a
b
 [] 
nil
 [=[]=] 
 [==[]==] 
 [[]] 
```

一个字符串可以由任意级别的正方括号开始，在遇到第一个同级别的反方括号时结束。整个词法分析过程将不受换行符限制，不处理任何转义符，并且会忽略掉任何不是同级别反方括号的括号。这种方式描述的字符串可以包含任何东西，当然同级别的反方括号除外。比如，上例中s4在遇到第一"]]"时，就已经结束了，因此第二个“]]’会引发程序异常。

此外，Lua 的字符串是一个不可改变的值，不能像在 c 语言中直接修改字符串的某个字符，而只能创建一个新的字符串。Lua 也不能通过下标来访问字符串中的某个字符。

Lua 字符串一般都会经历一个“内化”（intern）的过程，即两个完全一样的 Lua 字符串在 Lua 虚拟机中只存储一份。每一个 Lua 字符串在创建时都会插入到 Lua 虚拟机内部的一个全局的哈希表中。 这意味着

- 创建相同内容的 Lua 字符串并不会引入新的动态内存分配操作，所以相对便宜（但仍会有查询全局哈希表的开销），

- 相同内容的 Lua 字符串不会占用多份存储空间
- 已经创建好的 Lua 字符串之间进行相等性比较时是 O(1) 时间度的开销，而不是通常见到的 O(n).



### table（表）

在Lua中，Table是一个抽象的“关联数组”。“关联数组”是一种具有特殊索引方式的数组，它的索引通常是string或者number类型，但也可以是除nil以外的任意类型。

```lua
local corp = {
    web = 'www.google.com',  -- 索引为字符串。key ="web",value = "www.google.com"
    telephone = '12345678',  -- 索引为字符串。key = "telephone",value = "12345678"
    staff = {"jack","Scott","Gary"}, -- 索引为字符串。key = "staff", value是一个表
    100876,  -- 相当于 [1] = 100876，此时索引为数字。key = 1, value = 100876
    100191,  -- 相当于 [2] = 100191，此时索引为数字。key = 2, value = 100191
    [10] = 360,  -- 直接给出数字索引。key = 10, value = 360
    ["city"] = "BeiJing",  -- 索引为字符串。key = "city", value = "BeiJing"
}

print(corp.web)
print(corp["telephone"])
print(corp[2])
print(corp["city"])
print(corp.staff[1])
print(corp[10])
```

在内部实现上，table通常实现为一个哈希表、一个数组或两者的组合。具体实现为何种形式，依赖于table中的key-value的分布特点。



### funcation（函数）

在 Lua 中，函数也是一种数据类型，函数可以存储在变量中，可以通过参数传递给其他函数，还可以作为其他函数的返回值。

```lua
local function foo()
    print("in the function")
    local x = 10
    local y = 20
    return x,y
end

local a = foo  -- 把函数赋给变量
print(a)
print(foo)
```

有名函数的定义本质上是匿名函数对变量的赋值。如下示例：

```lua
function foo()
end

-- 等价于
foo = function()
end
```

类似地，

```lua
local function foo()
end

-- 等价于

local foo = function()
end
```



## 函数

### 函数的定义

在定义函数要注意几点：

- 名字的可读性。即利用名字来解释函数、变量的目的，使人通过名字就能看出来函数、变量的作用 
- 每个函数的长度要尽量控制在一个屏幕内，一眼可以看明白。
- 让代码自己说话，不需要注释最好  



#### 语法

Lua 使用关键字 function 定义函数，语法如下：

```lua
function function_name(arc) -- arc 表示参数列表，函数的参数列表可以为空
  -- body
end
```

如上定义了一个名为function_name的全局函数。我们知道在Lua中，全局函数本质上就是将函数类型的数据赋给了一个全局变量，即上面的语法等价于

```lua
function_name = function(arc)
    -- body
end
```



#### 局部函数

由于全局变量会污染全局名字空间，同时也有性能损耗（即查询全局环境表的开销），因此我们应当尽量使用“局部函数”，其语法与全局函数是类似的，只是开头加上 local 修饰符：  

```lua
local function function_name(arc)
  -- body
end
```



#### 函数 & Lua Table

由于Lua中的函数定义等价于变量赋值，因此我们也可以把函数名替换为 Lua 表中的某个字段。例如：

```lua
function foo.bar(a,b,c)
    -- body
end
```

如上我们把一个函数类型的数据赋给了 foo 表的 bar 字段。换言之，上面的定义等价于

```lua
foo.bar = function(a,b,c)
    print(a,b,c)
end
```

对于这种形式的函数定义，不能使用 local 修饰符，因为不存在定义新的局部变量。



### 函数的参数

#### 按值传递

在Lua中，函数的参数大部分是按值传递的。按值传递就是调用函数时，实参将它的值通过赋值运算传递给形参，之后形参的改变和实参就没有关系了。在这个过程中，实参是通过它在参数中的位置与形参匹配起来的。

```lua
local function swap(a,b)
    local temp =a 
    a = b
    b = temp
    print(a,b)
end

local x = "hello"
local y = 20
print(x,y)
swap(x,y)
print(x,y)
```

运行结果：

```lua
hello	20
20	hello
hello	20
```

在调用函数的时候，如果实参的个数和形参的个数不同时，Lua 会自动调整实参个数。调整规则为：若实参的个数大于形参的个数，按从左向右的顺序匹配，多余的参数被忽略；如果参数的个数小于形参的个数，按从左向右的顺序，没有被实参初始化的形参会被初始化为nil。

```lua
local function fun1(a,b)
    print(a,b)
end

local function fun2(a,b,c,d)
    print(a,b,c,d)
end

local x = 1
local y = 2
local z = 3

fun1(x,y,z)
fun2(x,y,z)
```

运行结果：

```lua
1	2
1	2	3	nil
```



#### 可变长参数

在上面的示例中，函数的参数都是固定的，其实Lua还支持可变长参数。如果形参为`...`，则表示该函数可以接受不同长度的参数。在访问可变长参数的时也要使用`...`。

```lua
local function func(...)                   -- 新参为...，表示函数使用可变长参数。
    local temp = {...}                     -- 访问可变长形参时也要使用...
    local ans = table.concat(temp, " ")    -- 使用table.concat库函数把数组内容使用“ ”拼接成字符串。
    print(ans)
end

func(1,2)
func(1,2,3,4)
```

运行结果：

```lua
1 2
1 2 3 4
```

需要注意的是，LuaJIT2尚不能JIT编译这种可变长参数的用法，只能解释执行。因此对性能敏感的代码，应当避免使用这种形式。



#### 具名参数

Lua还支持通过形参名来指定其实参，这个时候要把所有的实参组织到一个table中，并将这个table作为唯一的实参传给函数。

```lua
local function change(arg)
    arg.width = arg.width * 2
    arg.height = arg.height * 2
    return arg
end

local rectangle = { width = 20, height = 15}
print("before change:", "witdth =", rectangle.width, "height =", rectangle.height)

rectangle = change(rectangle)
print("after change:", "witdth =", rectangle.width, "height =", rectangle.height)
```

运行结果：

```lua
before change:	witdth =	20	height =	15
after change:	witdth =	40	height =	30
```



#### 按引用传递

当函数的参数是table类型时，传递进来的是实际参数的引用。此时在函数内部对该table所作的修改，会直接对调用者所传递的实际参数生效，而无需自己返回结果和让调用者进行赋值。

```lua
function change(arg):
    arg.width = arg.width
    arg.height = arg.height
end

local rectangle = { width = 20, height = 15}
print("before change:", "witdth =", rectangle.width, "height =", rectangle.height)

change(rectangle)
print("after change:", "witdth =", rectangle.width, "height =", rectangle.height)
```

运行结果：

```lua
before change:	witdth =	20	height =	15
after change:	witdth =	40	height =	30
```



#### 小结

在常用的基本类型中，除了table是按地址传递类型外，其它的都是按值传递。用全局变量来代替函数参数的不好编程习惯应该被抵制，良好的编程习惯应该是减少全局变量的使用。



### 函数的返回值

Lua 具有一项与众不同的特性，允许函数返回多个值，即函数可以有多个返回值。在Lua 的库函数中，有一些就是返回多个值的。



#### 示例1：库函数  string.find

例如，库函数  string.find  ，在源字符串中查找目标字符串，若查找成功，则返回目标字符串在源字符串中的起始位置和结束位置的下标。

```lua
local s,e = string.find("hello world", "llo")
print(s,e)
```

运行结果：

```lua
3	5
```

返回多个值时，值之间用 “,” 隔开。



#### 示例2：实现两个变量交换值  

定义一个函数，实现两个变量交换值

```lua
local function swap(a,b)
    return b,a
end

local x = 1
local y = 20
x, y = swap(x,y)
print(x,y)
```

运行结果：

```lua
20   1
```



#### 示例3：接收返回值的变量个数 vs 函数返回值的个数

当 函数返回值的个数 和 接收返回值的变量个数 不一致时，Lua 也会自动调整参数个数。调整规则为： 若 返回值个数 大于 接收变量的个数，多余的返回值会被忽略掉； 若 返回值个数 小于 接收变量的个数，从左向右，没有被返回值初始化的变量会被初始化为 nil。

```lua
function init()
    return 1, 'lua'
end

x = init()
print(x)

x, y, z = init()
print(x,y,z)
```

运行结果：

```lua
1
1 lua nil
```



#### 示例4：有多个返回值的函数 vs 列表表达式

当函数有一个以上的返回值，且函数调用不是一个列表表达式的最后一个元素，那么函数调用只会产生一个返回值, 也就是第一个返回值。

```lua
local function init()
    return 1,'lua'
end

local x, y, z = init(), 2
print(x,y,z)

loca a, b, c = 2, init()
print(a,b,c)
```

运行结果：

````lua
1 2 nil
2 1 lua
````

函数调用的实参列表也是一个列表表达式。我们看下面的例子：

```lua
local function init()
    return 1, 'lua'
end

print(init(),2)
print(2,init())
```

运行结果：

```lua
1 2 nil
2 1 lua
```



#### 示例5：多个返回值 & 只取一个值

如果你确保只取函数返回值的第一个值，可以使用括号运算符，例如：

```lua
local function init()
    return 1, 'lua'
end

print((init()),2)
print(2,(init()))

x,y,_ = 2,init()
print(2,1)
```

运行结果：

```lua
1 2
2 1
2 1
```

需要注意的是，如果实参列表中某个函数会返回多个值，同时调用者又没有显式地使用括号运算符来筛选和过滤，则这样的表达式是不能被 LuaJIT 2 所 JIT 编译的，而只能被解释执行。



### 全动态函数调用

#### 简介

全动态函数调用，是指调用一个回调函数，并把一个数组作为这个回调函数的参数。

```lua
local args = {...} or {}
method_name(unpack(args, 1, table.maxn(args)))
```

如果可以确定实参table中没有nil，则可以简化为：

```lua
method_name(unpack(args))
```



#### 使用场景

通常在以下场景中，我们使用回调函数：

- 要调用的函数，其参数是未知的；

- 函数的实参类型和个数也是未知的；

我们看如下伪代码：

```lua
add_task(end_time, callback, params)

if os.time() >= endTime then
    callback(unpack(params, 1, table.maxn(params)))
end
```

需要注意的是，内建函数unpack还不能被 LuaJIT 所 JIT 编译，因此这种用法总是会被解释执行。对性能敏感的代码应避免这种用法。



#### 使用示例

```lua
local function run(x,y)
    print('run', x, y)
end

local function attack(targetId)
    print('targerId', targetId)
end

local function do_action(method, ...)
    local args = {...} or {}
    method(unpack(args, 1, table.maxn(args)))
end

do_action(run,1,2)
do_action(attack,1111)
```

运行结果：

```lua
run 1 2
targetId 1111
```
